Description

An implementation of the Either monad, but with an emphasis on being compatible with existing Erlang code. Hence using {ok, ...} and {error, ...} instead of Right and Left (Right is success, Left is failure, traditionally). Otherwise, nomenclature should be identical to Haskell.
